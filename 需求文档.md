### 2. 分层职责说明

1. **Rust (Tauri 后端核心)**

   * 系统托盘常驻图标，程序最小化后仅保留托盘图标；
   * 定时任务引擎：1 小时宠物唤醒、3 天到期任务轮询检测；
   * SQLite 本地存储：全部任务、宠物配置、提醒规则本地持久化，无需依赖远端服务；
   * 系统原生窗口：宠物弹窗、到期提醒弹窗置顶展示；
   * 硬件能力封装：麦克风录音、文件读写、开机自启、窗口显示 / 隐藏控制；
   * 提供 Invoke 接口给 Vue 前端调用，隔离系统底层操作。
2. **Vue3 前端业务层**

   * 页面 UI、任务 CRUD、AI 输入交互、宠物动画渲染；
   * Pinia 全局管理任务数据、提醒配置、宠物开关；
   * Lottie 渲染卡通宠物跳舞动画；
   * 语音转文字、文本提交 AI 解析表单；
   * 今日待办自动计算、任务筛选、优先级展示。
3. **AI 语义解析层**

   输入文本 / 语音转文字后，NLP 抽取 4 类关键信息：任务内容、截止时间、重要程度、预估工作量，结构化生成待办数据。

## 二、产品五大核心功能技术实现方案

### 功能 1：语音 / 文字输入，AI 自动拆解生成结构化任务

#### 流程

1. 用户交互入口（托盘悬浮面板）

   * 文字输入框；
   * 录音按钮，调用 Tauri 麦克风 API 录制语音；
2. 语音处理链路

   录音文件 → Rust 转 wav → 调用语音转文字接口 → 输出纯文本；
3. NLP 语义抽取核心输出字段

   表格

   | 抽取字段    | 说明                          |
   | ----------- | ----------------------------- |
   | taskContent | 任务详情                      |
   | deadline    | 计划完成时间                  |
   | priority    | 重要程度：紧急 / 高 / 中 / 低 |
   | workload    | 工作量（1-5 级，对应耗时）    |
4. 自动入库：解析完成直接生成待办任务存入本地 SQLite。

#### 技术实现区分

* 云端方案：请求大模型 API，Prompt 约束固定 JSON 结构化返回，开发成本低；
* 本地离线方案：Rust 加载轻量分词 + 实体抽取模型，无网络也可用，包体积增大。

### 功能 2：到期 3 天任务自动提醒

1. Rust 定时轮询（后台常驻，无需前端窗口打开）
   * 每 30 分钟执行一次任务扫描；
   * 筛选规则：当前日期 ≤ 截止日期 ≤ 当前日期 + 3 天、未完成任务；
2. 提醒表现形式
   * 托盘图标小红点角标，展示临近到期任务总数；
   * 弹出置顶系统弹窗，列出所有即将到期任务；
   * 打开主面板时，临近 3 天任务置顶标黄高亮；
3. 数据驱动：任务表增加 `isRemind`字段，支持单任务关闭提醒。

### 功能 3：自动生成今日待办清单

#### 自动计算逻辑（前端 + Rust 双重计算）

1. 筛选条件：

   * 截止日期≥今日；
   * 未完成；
   * 优先级高 / 紧急优先；
2. 智能排序规则：

   紧急任务 > 3 天内到期任务 > 今日工作量大任务 > 普通任务；
3. 展示能力：

   * 托盘悬浮面板首页直接展示今日待办；
   * 主页面独立「今日待办」标签页；
   * 支持手动拖拽调整，覆盖 AI 自动排序结果。

### 功能 4：卡通宠物每小时弹窗活动提醒

1. 定时调度（Rust 后台计时，不受前端窗口关闭影响）
   * 全局计时器，每小时触发一次；
   * 支持用户自定义开关：开启 / 关闭宠物提醒、自定义提醒间隔（30min/1h/2h）；
2. 宠物弹窗窗口特性
   * Tauri 独立无边框置顶小窗口；
   * Vue + Lottie 渲染连续跳舞动态动画；
   * 附带文字提示：“起身活动 5 分钟，多喝温水”；
   * 交互按钮：稍后提醒（延后 1 小时）、关闭本次弹窗、永久关闭宠物；
3. 持久化配置：用户开关、提醒间隔存入本地数据库，重启程序不重置。

### 功能 5：桌面常驻托盘小图标，默认最小化隐藏主窗口

1. Tauri 托盘核心能力
   * 程序启动后，主窗口自动隐藏，仅右下角托盘显示图标；
   * 托盘右键菜单：打开任务面板、开启 / 关闭宠物、设置、退出程序；

AI文本解析模块，要怎么接入大模型

# Tauri2.0+Vue3 待办工具 AI 文本解析模块接入大模型完整方案

目标：用户输入一段自由文本 / 语音转文字后的文本，大模型输出结构化 JSON（任务内容、截止时间、重要程度、工作量），存入本地 SQLite。

分 3 套接入方案： **公有云大模型 API（最快落地）、本地离线大模型（无网络可用）、企业私有大模型** ，同时配套 Tauri 前后端代码、Prompt、异常处理、安全优化。

## 一、统一需求：大模型固定输出结构

### 1. 标准返回 JSON 格式（强制约束，避免解析错乱）

json

```
{
  "taskList": [
    {
      "content": "完成Q3部门业绩报表",
      "deadline": "2026-06-30",
      "priority": "紧急/高/中/低",
      "workload": 1~5,
      "remark": "补充说明（可选）"
    }
  ]
}
```

* deadline：统一 `YYYY-MM-DD` 标准日期，无时间则返回空字符串；
* priority 四选一：紧急、高、中、低；
* workload：1=1 小时内，5 = 全天耗时；
* 一段文本可拆分多条任务，数组支持多任务。

### 2. 固定系统 Prompt（核心，保证输出稳定）

plaintext

```
你是任务提取助手，只输出JSON，禁止多余文字、解释、markdown。
输入一段用户待办描述，提取所有任务，按规则输出：
1. content：任务完整内容；
2. deadline：计划完成日期，只识别年月日，格式YYYY-MM-DD，无则为空；
3. priority：根据描述判定紧急/高/中/低；
4. workload：预估耗时1-5数字；
规则：
- 提到今天/明天/3天后自动换算对应日期；
- 带有尽快、马上、今晚标记为紧急；
- 复杂长期工作工作量5，简单小事工作量1；
输出严格遵循：{"taskList": [{}]}
```

## 二、方案 1：公有云 API 接入（推荐快速开发，豆包 / 通义千问 / 文心一言）

### 架构分工（Tauri 分层，不把密钥暴露前端）

> 关键： **密钥存 Rust 后端，Vue 前端只传文本，不携带 API Key** ，防止抓包泄露。

1. Vue3 前端：收集用户输入文本 → 调用 Tauri invoke 传给 Rust；
2. Rust (Tauri 后端)：内置 HTTP 客户端，携带密钥、Prompt 调用大模型接口；
3. Rust 接收模型返回 → JSON 校验清洗 → 返回结构化任务给前端；
4. 前端拿到结构化数据，直接入库生成待办。

### 1. Rust 侧封装通用大模型请求函数（以豆包为例）

rust

运行

```
// src-tauri/src/ai_client.rs
use reqwest::Client;
use serde_json::{json, Value};

// 大模型配置，可写入本地配置文件，用户设置面板修改
const API_KEY: &str = "sk-xxx";
const API_URL: &str = "https://ark.cn-beijing.volces.com/api/v3/chat/completions";
const MODEL_ID: &str = "doubao-pro";

pub async fn parse_task_text(user_input: &str) -> Result<Value, String> {
    let client = Client::new();
    let system_prompt = r#"你是任务提取助手，只输出JSON，禁止多余文字、解释、markdown。
输入一段用户待办描述，提取所有任务，按规则输出：
1. content：任务完整内容；
2. deadline：计划完成日期，只识别年月日，格式YYYY-MM-DD，无则为空；
3. priority：根据描述判定紧急/高/中/低；
4. workload：预估耗时1-5数字；
规则：
- 提到今天/明天/3天后自动换算对应日期；
- 带有尽快、马上、今晚标记为紧急；
- 复杂长期工作工作量5，简单小事工作量1；
输出严格遵循：{"taskList": [{}]}"#;

    let req_body = json!({
        "model": MODEL_ID,
        "messages": [
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_input}
        ],
        "temperature": 0.1 // 低随机性，保证结构化输出稳定
    });

    let resp = client.post(API_URL)
        .bearer_auth(API_KEY)
        .json(&req_body)
        .send()
        .await
        .map_err(|e| format!("网络请求失败：{}", e))?;

    let res_json: Value = resp.json().await.map_err(|e| format!("解析返回失败：{}", e))?;
    // 提取模型返回内容
    let raw_content = res_json["choices"][0]["message"]["content"]
        .as_str()
        .ok_or("大模型无返回内容")?;
  
    // 清洗：去除markdown ```json 标记
    let clean_str = raw_content.replace("```json", "").replace("```", "").trim();
    // 转为标准任务JSON
    let task_data: Value = serde_json::from_str(clean_str)
        .map_err(|e| format!("模型输出JSON格式错误：{}", e))?;
    Ok(task_data)
}
```

async fn ai_parse_task(text: &str) -> Result<serde_json::Value, String> {
    ai_client::parse_task_text(text).await

### 3. Vue3 前端调用示例

vue

```
<script setup>
import { ref } from 'vue'
import { invoke } from '@tauri-apps/api/core'

const inputText = ref('下周三之前完成需求文档，这个工作比较复杂，尽快做完')
const taskList = ref([])
const loading = ref(false)

async function handleAiParse() {
  loading.value = true
  try {
    // 只传文本，密钥完全在Rust后端，前端无敏感信息
    const res = await invoke('ai_parse_task', { text: inputText.value })
    taskList.value = res.taskList
    // 直接批量插入本地SQLite
    await saveTaskToDb(taskList.value)
  } catch (err) {
    console.error('AI解析失败', err)
  } finally {
    loading.value = false
  }
}
</script>
```

### 4. 配套优化点

1. **密钥管理**

   * 不硬编码密钥：程序设置面板让用户填入自己的 API Key，存入本地 `app_config` SQLite 表；
   * 加密存储密钥（Rust 使用 ring 库简单加密，避免明文）；
2. **网络异常降级**

   * 无网络时禁用 AI 解析，提示用户手动填写任务；
   * 增加超时 10s，防止接口卡死；
3. **限流控制**

   * 前端增加防抖，1s 内不可重复请求；
   * 后端增加单次文本长度限制（500 字），减少 token 消耗；
4. **日期自动换算**

   模型无法识别相对日期时，Rust 层补充时间计算兜底（今天、明天、3 天后）。
